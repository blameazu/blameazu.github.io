題目
#

TOJ 772 神魔之塔

題敘又臭又長(但比賽的時候認真看題敘也是一門技巧)

以下是簡短的題敘

給你 $m$ 個元素的集合 $C$

定義

$$ v[i] = \prod^{i-1}_{j = 1} gcd(i, j)^j[gcd(i, j) \in C], \forall 1 \le i \le n $$

給 $q$ 筆操作，每次給 $l, r$

詢問

$$ \prod^{r}_{k = l} v[k] \pmod{998244353} $$
  • $1 \le n, q \le 10^5$
  • $1 \le m \le 100$

先來拆解問題
#

首先我們發現，我們沒有修改到 $v[i]$ 的值吧?

所以對於 $Q$ 筆詢問

我們可以先計算 $pre_i \forall 1 \le i \le n$

那麼每次的答案只要輸出 $\frac{pre_r}{pre_{l-1}} \pmod{998244353}$

也就是 $pre_r \times (pre_{l-1})^{998244353-1} \pmod{998244353}$

右邊那個是用費馬小定理求的逆元

現在問題剩下了什麼?
#

發現我們只要求出每個 $v[i]$

那我們就可以花 $O(n)$ 的時間預處理

花 $O(q \log V)$ 的時間回答?

ps. $V$ 是值域，因為要做快速冪

好好好

那我們要怎麼求每個 $v[i]$ 呢?

數學風暴
#

首先可以發現數學式子長太醜了

我們一定不可以枚舉每個 $j$ 去算!

這樣 $O(n^2)$

$$ v[i] = \prod^{i-1}_{j = 1} gcd(i, j)^j[gcd(i, j) \in C], \forall 1 \le i \le n $$

轉換思維，我們換成枚舉公因數 $d = gcd(i, j)$

式子長怎樣?

$$ v[i] = \prod_{d \mid i, d \in C} d^{\displaystyle \sum^{i-1}_{j = 1} j[d \mid j]}, \forall 1 \le i \le n $$

蛤? 右上角那三小?

$$ \sum^{i-1}_{j = 1} j[d \mid j] $$

這樣我們還不是要枚舉 $j$ …

先別急! 式子還沒推導完

我們再將式子變一下

$$ \begin{aligned} & \sum_{j=1}^{i-1} j[d \mid j] \\\\ & = \sum_{k = 1}^{\frac{i}{d}-1} (k \times d)[gcd(k, \frac{i}{d}) = 1] \\\\ & (\text{將 } d \text{ 提出來}) \\\\ & = d \sum_{k = 1}^{\frac{i}{d}-1} k [gcd(k, \frac{i}{d}) = 1] \\\\ & (\text{定義 } L = \frac{i}{d}) \\\\ & = d \sum_{k = 1}^{L-1} k [gcd(k, L) = 1] \end{aligned} $$

那麼現在題目又變了!

轉成

解 $1 \le k \lt L$ 且 $gcd(k, L) = 1$ 的數的總和

這個我們可以預處理好嗎?

好熟悉喔
#

擁有驚人的觀察，會發現 上面導出的問題

好像是歐拉函數 $\phi (i)$ 喔

$$ \phi (1) = 1 $$$$ \phi (i) = \sum_{1 \le j \le i} 1 [gcd(i, j) = 1], \forall i \ge 2 $$

現在 $1$ 改成了 $j$，我們就不會算了

嗎?

What The F
#

我們可以知道一個定理

$$ gcd(a, b) = gcd(a, b-a) $$

我們再擁有億點點驚人的觀察

發現 $gcd(k, L) = gcd(L-k, L) = 1$

也就是對於小於 $L$ 且與 $L$ 互質的數字是成對出現的!!!

且每一對加起來 $k + (L-k) = L$ 剛好是 $L$

那我們總共有幾對與 $L$ 互質的質數對呢?

$$ \frac{\phi(L)}{2} $$

耶，所以我們把東西代回好遠古的式子裡面

$$ \sum_{k = 1}^{L-1} k [gcd(k, L) = 1] = L \times \frac{\phi(L)}{2} $$$$ \sum_{j=1}^{i-1} j[d \mid j] = d \times (L \times \frac{\phi(L)}{2}) $$

把 $L$ 變回來

$$ \sum_{j=1}^{i-1} j[d \mid j] = d \times (\frac{i}{d} \times \frac{\phi(\frac{i}{d})}{2}) $$$$ \sum_{j=1}^{i-1} j[d \mid j] = \frac{i \times \phi(\frac{i}{d})}{2} $$

再再再代回恐龍時代的原式

$$ v[i] = \prod_{d \mid i, d \in C} d^{\displaystyle \sum^{i-1}_{j = 1} j[d \mid j]}, \forall 1 \le i \le n $$$$ v[i] = \prod_{d \mid i, d \in C} d^{\displaystyle \frac{i \times \phi(\frac{i}{d})}{2}}, \forall 1 \le i \le n $$

我們已知 $d \in C$

所以我們只要暴力枚舉 $m = 100$ 個

我們就可以花費 $O(n \times m)$ 的時間預處理出 $v[i]$ 陣列了!

小 bug
#

你發現

$L = 1 = \frac{i}{d}$ 會有問題欸

因為 $\phi(1) = 1$，他算到了自己

但是我們的 $k$ 要小於 $L$

所以我們就把這個情況判掉

簡單方法是直接設 $\phi (1) = 0$

實作
#

Spoiler

全域
# 

const int mod = 998244353;

快速冪
#

你要先會快速冪(fast pow)

不然你連逆元都不會求

遞迴版本

int fpw(int a, int b) {
    if(b == 0) return 1;
    int re = fpw(a, b>>1);
    if(b&1) return 1ll*re*re%mod*a%mod;
    return 1ll*re*re%mod;
}

迴圈版本

int fpw(int a, int b) {
    int re = 1;
    while(b) {
        if(b&1) re = 1ll*re*a%mod;
        a = 1ll*a*a%mod;
        b >>= 1;
    }
    return re;
}

求 phi
#

你可以用 $O(n \sqrt{V}) = O(n \sqrt{n})$ 的時間求 $\phi(i)$，沒人阻止你

但你也可以很強的發現 $\phi(i)$ 是積性函數

所以可以用線性篩做到 $O(n)$ 預處理

const int N = 1e5;
vector<int> phi(N+5), isprime(N+5, 1);
vector<int> primes;
for(int i = 2; i <= N; i++) {
    if(isprime[i]) {
        primes.push_back(i);
        phi[i] = i-1;
    }
    for(auto &d : primes) {
        if(d * i > N) break;
        isprime[d*i] = 0;
        if(i%d == 0) phi[i*d] = phi[i] * d;
        else phi[i*d] = phi[i] * phi[d];
    }
}

計算 $v[i]$
# 

for(int i = 1; i <= n; i++) {
    v[i] = 1;
    for(auto &d : C) if(d != 1 && i%d==0)
        v[i] = 1ll*v[i] * fpw(d, (1ll*i*phi[i/d])/2)%mod;
