前言
#

這篇來介紹怎麼使用演算法來解決 最大流問題

先備知識
#

知道 flow 是啥

知道 flow 在圖上大概長怎樣

管道逆流
#

等等會提到一個很關鍵的步驟是管道逆流

也就是說當有一條水管 $u \to v$

被流過去 $f$ 這條水量時

我們允許 $v \to u$ 多逆流回去 $f$

而詳細的操作應該是

當有一條容量為 $C$ 的水管 $u \to v, v \to u$

那麼當 $u \to v$ 是 $A$，$v \to u$ 是 $B$ 時

當 $u \to v$ 多流 $x$ 的時候， $v \to u$ 應該就要減少流量 $x$

這樣就確保 $\vert A - B \vert \le C$

Fulkerson Ford 方法
#

這是一種方法，不是一種演算法!

Fulkerson Ford 告訴你，當你每次找一條增廣路(邊上流量都是正數的流)

假設該條的流量瓶頸(最小的那一個) 為 $f_i$

那麼最大流量的值就為 $\sum f_i$

但前提條件是我們要將每一條找出來的增廣路做反向邊

也就是說當我們走過一條 $(u, v)$，流量 為 $f$ 的邊

我們就將那條邊做逆推流量 $f$ 的動作

為什麼要這麼做呢?

我們可以感性理解一下，把它想像成一種 greedy 的體現方法

當我走過那條邊的時候，我可以反著逆回去，把水流推回去

也就是說我們允許之後可以將我們當前的操作逆轉回去

好那麼這樣我們就講完 Fulkerson Ford 的觀念了!

時間複雜度的話 $O(F \cdot \text{搜尋時間})$

$F$ 是流大小，搜尋時間通常為 $E$ 因為用 BFS、DFS

Edmonds Karp 演算法
#

這是基於剛剛所說的 Fulkerson Ford 的一種實作版本

搜尋方法採用 BFS

簡單的做法就如下:

  1. 用 BFS 找一條最短的增廣路徑($F > 0$)
  2. 將他加到答案中，(因為答案等於 $\sum F$)
  3. 建整條流的反向邊

這樣就做完了!

複雜度為 $O(V E^2)$

簡單來講就是每條邊最多會被當作 $V$ 次的瓶頸邊

總共有 $E$ 條邊，所以就是 $O(VE)$

然後再乘上每次搜尋的時間 $O(V + E) = O(E)$

所以是 $O(V E^2)$

至於為什麼每條邊最多只會被當 $V$ 次瓶頸邊呢?

我們不妨先假設整張圖權重皆為 $1$

接著我們可以先簡單地知道一下

我們會走到 $u \to v$ 的這條反向邊

必定是 $dis’[u] > dis[v] + 1$

也就是說當 $dis[v] = dis[u] + 1$

所以是 $dis’[u] > dis[u] + 2$

所以最多只會增長 $V$ 次(因為 dis 最多 $N$ 個)

有權重應該差不多

實作
#

對於演算法來講，實作肯定是最重要的部分

建邊
#

首先我們要先知道怎麼建邊

為了應對重邊的問題，所以我們就不採用鄰接矩陣喔

假如邊 $E$ 為 $u \to v$，容量為 $C$，流量為 $0$

那麼定義 邊 $E \oplus 1$ 為 $v \to u$，容量為 $C$，流量為 $0$

這樣定義有一個好處

當我們在 $A$ 這個點的串接矩陣中塞的是邊的編號

那麼當我們在逆推的時候會特別輕鬆就知道反向邊是哪一條

vector<vector<int> > G(n+1); // 圖，串接矩陣，裡面塞的是邊的編號
vector<array<int, 3> > edges; // 邊(u->v), v, c, f


void add_edge(int u, int v, int c) {
    edges.push_back({v, c, 0});
    G[u].push_back(edges.size()-1);
    edges.push_back({u, 0, 0});
    G[v].push_back(edges.size()-1);
}

BFS
# 

vector<int> dis(n+1); // 從 s 來的最大流量是多少
vector<int> fr(n+1); // 紀錄用哪條邊來的

int bfs(int s, int t) {
    for(int i = 1; i <= n; i++)
        dis[i] = -inf; // inf 是無限大，看你爽設什麼就設什麼
    dis[s] = inf;
    queue<int> qq;
    qq.push(s);
    while(qq.size()) {
        auto u = qq.front(); qq.pop();
        for(auto &id : G[u]) {
            auto [v, c, f] = edges[id];
            if(min(dis[u], c-f) > 0) { // 走過去流量需要 > 0
                dis[v] = min(dis[u], c-f);
                fr[v] = id;
                qq.push(v);
            }
        }
    }
    if(dis[t] == -inf) return -1; // 走不到 t
    return dis[t];
}

主功能(逆推邊)
# 

int ans = 0; // 最大流

while(true) {
    int re = bfs(s, t);
    if(re == -1) break;
    ans += re;
    int now = t;
    while(now != s) {
        auto &[v, c, f] = edges[fr[now]]; // 去的邊
        auto &[u, C, F] = edges[fr[now]^1]; // 反向邊
        f += re;
        F -= re;
        now = u;
    }
}

最小鏈覆蓋與最大流
#

當每條鏈的節點不可重複時

則最小鏈覆蓋 = $N-\text{最大流}$

當節點可重複時

最小鏈覆蓋也是 = $N-\text{最大流}$

但是那個最大流的圖要重構一下

當一個 $a$ 點間接 or 直接可以走到 $b$

則建邊 $a \to b$
