ABC 168 E

題目
#

ABC168 E - ∙ (Bullet)

今天有 $N$ 個沙丁魚要放進去冰箱裡面

定義 丁魚 $i$ 與 丁魚 $j$ 會吵架是 $A_i \times A_j + B_i \times B_j = 0$

你需要求出有多少種放法，使得放在冰箱裡的丁魚們之間無任意兩者會吵架

冰箱裡至少需要放 $1$ 隻

$1 \le N \le 2 \times 10^5$

$-10^{18} \le A_i, B_i \le 10^{18}$

題解
#

觀察
#

這題假如有學過高中數學的向量單元，應該可以很輕易地看出來 $A_i \times A_j + B_i \times B_j$ 很像內積

所以我們可以假設每個沙丁魚的 $(A_i, B_i)$ 為二維平面上面的座標向量

那麼我們就可以知道，當兩隻沙丁魚會吵架

那麼必定是他們與原點的方向向量垂直!!!

轉換為數學模型
#

所以我們就先把相互垂直的直線綁在一起看!

我們會發現除了這兩個互相垂直的直線

其他的直線與他們是沒有關聯的

所以答案就是每一組的答案相乘

那麼我們仔細看每一組自己本身的答案是多少

假設直線 $L$ 擁有 $A$ 個點

直線 $L’$ 擁有 $B$ 個點

那麼答案很明顯是

$$ (C^A_1 + C^A_2 + \ldots + C^A_A) + (C^B_1 + C^B_2 + \ldots + C^B_B) + 1 $$

也就是直線 $L$ 上取 $1, 2, \ldots A$ 個點以及直線 $L’$ 上取 $1, 2, \ldots B$ 個點，以及兩條線都不取!

所以整理出來的式子應該是

$$ ans = \Pi_{\{A \to count(L), B \to count(L'), \forall (L, L')\}} (2^A + 2^B -1) $$

蛤? 到這邊還是錯的欸?

我的答案怎麼多一個?

因為我們把什麼都不取的情況算進去了!

所以我們最終要把答案 $-1$

細節
#

到這邊開開心心地做了一個解出來，傳上去發現吃 WA 了

為甚麼呢?

我們沒有想到很關鍵的 Edge Case!

當 $(A_i, B_i) = (0, 0)$ 的時候!!!

這樣他與所有的點內積皆為 $0$ !!!

所以我們把它特判掉就能開心 AC 了

實作
#

Spoiler

在記錄直線的時候我是直接採用 map<pair<int, int>, pair<int, int> >

前面紀錄的是單位向量，接著後面的是 $(\text{這個單位向量上的點}, \text{與這個單位向量垂直的點})$

實作上可以採用將三四象限的點($A < 0$) $A, B$ 皆乘上 $-1$ 轉到一二象限

然後再判斷在哪個象限去紀錄垂直

但因為要特判 $(0, 1), (0, -1)$ 這種 case

所以我寫一個比較 general 的方法

map<pair<int, int>, pair<int, int> > mp;
int a, b; cin >> a >> b;
int g = gcd(a, b); 
if(g != 0) { // g == 0 -> RE
    a /= g;
    b /= g;
}
if(mp.count({-a, -b})) mp[{-a, -b}].first++;
else if(mp.count({b, -a})) mp[{b, -a}].second++;
else if(mp.count({-b, a})) mp[{-b, a}].second++;
else mp[{a, b}].first++;

根據高中數學可以得知，與向量 $(a, b)$ 垂直的向量有兩個 $(-b, a), (b, -a)$

所以我只要先判他之前有沒有被放進去過 map 就可以了 omob

最後就計算答案

int ans = 1, plus = 0;
for(auto &[a, b] : mp) {
    if(a.first == 0 && a.second == 0) {
        plus = b.first;
        continue;
    }
    int now = ((fpw(2, b.first) + fpw(2, b.second) -1 + mod)%mod);
    ans = (ans * now)%mod;
}
cout << (ans-1+mod+plus)%mod << '\n';

plus 拿來特判 $(0, 0)$ 的 case

然後 fpw 是快速冪，相信大家都會刻

結語
#
