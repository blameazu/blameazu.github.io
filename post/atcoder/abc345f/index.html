ABC345 F - Many Lamps

2025-12-17 Atcoder Solution

題目

今天有一個簡單無向圖

$N$ 個節點，$M$ 條邊

第 $i$ 條邊連接點 $v_i$ 與 $u_i$

每個節點上都有一盞燈，一開始每盞燈都是關著的

你可以對每一條邊選擇進行操作

  • 將第 $i$ 條邊的左右兩節點上面的燈狀態反轉，開 $\to$ 關，關 $\to$ 開

問你是否可以做到最後有 $K$ 個燈是開著的，假如有請構造出解

題解

性質

我們可以先觀察到一個很明顯的性質

當你對一條邊進行操作時，會有三種狀況

  1. 當左右兩點都未被點亮，則反轉後點亮的燈數 $+ 2$
  2. 當左右兩點都被點亮，則反轉後點亮的燈數 $- 2$
  3. 當左右兩點有一點被點亮、一點沒點亮，則反轉後點亮的燈數不變

這可以使我們發現 $K$ 為偶數才有可能有解

將圖轉成樹

我們發現圖上反轉操作真的有點麻煩

我們是否可以將圖用樹來看呢?

(痾 以下不失一般性的假設圖連通)

我們發現，當我們構一棵從節點 $1$ 出發的樹

那麼我們可以盯著它，使用瞪眼法

發現有一個超酷的性質(要發現才會發現，所以這步 = 通靈)

當你從一個子樹的匯集點，也就是說樹的內部節點

有一個性質

我們能使子樹內的節點都為點亮的燈，但子樹的根無法確定狀態

這應該很明顯，假設我們現在子樹的根連接著很多個已經做好的子樹

那麼我們沒有點亮的燈只會處於跟根節點相連的那些子樹的根

這也許沒到很明顯，我們可以簡單的舉個例子

上圖是根結點與 一個 沒點亮的節點連接

那麼我們可以將那條邊反轉，接著整顆樹就變成全點亮

上面這張圖是說根結點與 兩個 沒點亮的節點連接

那麼我們也是將兩條邊反轉，那麼除了根結點，其他的節點皆被點亮

所以我們可以得出一個結論

  • 當根結點與 奇數個 未點亮的節點相連，則我們可以透過一系列操作，使得整棵樹皆為點亮的節點

  • 當根結點與 偶數個 未點亮的節點相連，則我們可以透過一系列操作，使得整棵樹除了根結點皆為點亮的節點

證明最優

蛤? 我們剛剛只導出了我們能構出一棵根結點未知然後其他節點被點亮的樹

這樣是最佳嗎???

會不會在其他情況下可以使得根結點也被點亮?

結論是沒有的，我們可以細說細說

當 $N$ 個點的樹，被經過一系列操作後

那麼點亮的個數一定為 $N$ 或者 $N-1$

那麼根據好久好久得出來的結論

經過一系列操作，點亮的燈數只可能為偶數個

再看一下 $N$ 與 $N-1$，發現當 $N-1$ 可以被操作出來，那麼那個肯定是最佳了，因為 $N$ 個的是奇數，無法被構出!!!

(以上不專業證明，但我覺得很好理解)

喔然後不連通的圖你可以稍微思考一下，其實每個連通塊的最佳性都一樣

K 個

蛤? 但題目要求我們要構出 $K$ 個欸

我們現在全部都點亮了欸?

挖，那你肯定沒有記起來我們好久好久之前導出的一個性質

每次點亮個數一定是 +2 or +0 or -2

也就是說，我們可以透過中間值定理(x)

發現我們能到達最大值 $N$ or $N-1$ 的話

肯定會經過 $K$!!!

所以我們只要在構到 $K$ 的情況下結束程式就行!!

構造方法

聽了這麼多，根本沒教怎麼構嘛

年輕人，你太急了

剛剛說到我們可以將圖轉成樹嘛

然後又講到我們應該要盡可能的把未點亮的聚集到根結點

那麼我們可以發現

簡單的 DFS，當你為目前DFS的子樹的根結點，且連接的點為未點亮，則我們就反轉邊，使它點亮

那麼簡單的遞迴就可以做出來了 omob

最後判斷 $K$ 是否有做到即可

code

#include <bits/stdc++.h>
using namespace std;
signed main() {
	ios::sync_with_stdio(false), cin.tie(0);
	int n, m, k; cin >> n >> m >> k;
	vector<vector<pair<int, int> > > f(n+1);
	for(int i = 1; i <= m; i++) {
		int a, b; cin >> a >> b;
		f[a].push_back({b, i});
		f[b].push_back({a, i});
	}
	vector<int> tmp(n+1);
	vector<bool> vis(n+1);
	vector<int> ANS;
	int ans = 0;
	auto dfs = [&](auto dfs, int id) -> void {
		vis[id] = true;
		for(auto &[d, eid] : f[id]) if(!vis[d]) {
			dfs(dfs, d);
			if(tmp[d] == 0 && ans < k) {
				ans -= tmp[id];
				tmp[d] = 1;
				tmp[id] ^= 1;
				ans += tmp[id] + 1;
				ANS.push_back(eid);
			}
		}
	};
	for(int i = 1; i <= n; i++) if(!vis[i]) dfs(dfs, i);
	if(ans == k) {
		cout << "Yes\n";
		cout << ANS.size() << '\n';
		for(int i = 0; i < ANS.size(); i++) cout << ANS[i] << " \n"[i==ANS.size()-1];
	} else cout << "No\n";
}

複雜度為 $O(N+M)$
